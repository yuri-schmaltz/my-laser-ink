# Licenciamento e Componentes de Terceiros

## Objetivo
Manter clareza de licenças e obrigações de distribuição.

## Diretrizes
- Preferir MIT/BSD/Apache para código incorporado ao binário principal.
- Evitar incorporar código GPL no binário principal (pode impor copyleft no todo).
- Componentes LGPL devem ser isolados (ex.: serviço separado), quando aplicável.

## Terceiros (preencher)
- Laser Ink (base do app)
- MeerK40t (módulos/adapter, se usado)
- Bibliotecas de parsing (SVG/DXF/PDF)
- Renderização/GUI e dependências

## Checklist de release
- LICENSE(s) presentes
- NOTICE/THIRD_PARTY atualizado
- Fontes redistribuídas quando exigido
